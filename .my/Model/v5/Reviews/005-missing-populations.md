# Missing Populations and Edge Cases — Model V5

## TL;DR

- **AI-native users** (children, teens) don't experience the initial encounter the model requires; no pre-formed identity to threaten.
- **Non-knowledge work** (trades, manual labor, service) doesn't fit the model's framework; threat is economic, not identity-based.
- **Cognitive disabilities** absent. The model assumes users can integrate new information and assess their own limits.
- **Emotional or companionship use** has no pathway; the model assumes users want to do work better.
- **AI researchers** skip the testing phase; they understand the technology already.
- **Economic dependence on non-adoption** (human-only service providers) faces permanent threat, not temporary shock.
- **Pre-emptive ethical rejection** (before any testing) doesn't fit; the model assumes ethical concerns come after trying the tool.
- **Adoption Ceiling gap:** V5 removed the construct allowing bounded use. A person can fully understand AI yet only use it within limits; forced beyond, they destabilize.

## Findings

### 1. AI-native users (children and teens who first encounter AI before age 18)

The model requires Initial Encounter — the moment AI seems novel and threatening — as mandatory. An AI-native user never has this moment. A 16-year-old hasn't decided whether coding or writing is core to their sense of self. When they use AI for writing, it doesn't trigger Ego Shock (the defensive reaction when a tool threatens core skill) because there is no core skill yet. The model predicts threat triggers shutdown; this user has no identity to defend, so they bypass shock entirely and may land directly in Identity Expansion without resistance.

**Fix:** Add an alternate path for users who first encounter AI before forming domain identity. They should skip directly to Understanding (a stable state with an accurate model of what the tool can do) or Identity Expansion. Or explicitly state the model applies only to adults with pre-formed domain identity.

### 2. Non-knowledge work (trades, manual labor, service work)

The model uses knowledge-work language throughout: "skill," "craft," "expertise," "domain." These fit designers and programmers. An electrician has diagnostic skills, but they're physical and hands-on. The Task Delegation framework (Information, Task Execution, Creative, Autonomous Partnership) assumes the worker decides, analyzes, or creates. In knowledge work, Ego Shock hits as identity threat ("my skills are being replicated"). In non-knowledge work, it hits as economic threat ("I will be replaced"). The model doesn't separate them.

**Fix:** Define a separate Task Delegation frame for non-knowledge work, or add a note explaining how delegation levels and identity stakes map differently. At minimum, flag that shock in these domains is economic, not identity-based.

### 3. People with cognitive disabilities

The model assumes all users can integrate new information into existing mental models and accurately assess their own limits. Neither holds for everyone. A user with a cognitive disability (intellectual disability, working-memory deficit, processing-speed impairment) might use AI as a cognitive prosthetic — to fill a gap rather than to compete with existing skill. A person with dyslexia using AI to rewrite text is not in shock. The model assumes all AI use is capability-matching (you can already do the thing, and AI does it too); it has no state for capability-enabling (AI opens a domain you couldn't access before).

**Fix:** Add an optional Understanding state for accessibility use, or explicitly state the model applies only to users not requiring cognitive assistive technology.

### 4. Emotional and companionship use

The entire model assumes the use case is productivity-based. A user seeking emotional companionship or support from AI (a chatbot friend, a virtual mentor) doesn't fit any state. Identity Stakes don't apply. Task Delegation doesn't map. Shock doesn't trigger. Understanding requires the user to build an accurate mental model of the tool — but if the use is relational, the entire point is *not* treating it as a tool requiring calibration.

**Fix:** Acknowledge explicitly that emotional-companionship use is out of scope. Or add a separate fork from Initial Encounter that describes relationship-formation transitions.

### 5. AI researchers

The model assumes users lack insider knowledge. An AI researcher already has accurate mental models of what AI can and cannot do. They don't spend time testing because they already know the failure modes. They don't enter Defensive Resistance because they're in technical-assessment mode, not ego-defense mode. A researcher might jump directly to Ego Shock or skip it entirely and enter Ethical Integration.

**Fix:** Note that high-expertise users skip Defensive Resistance and may accelerate to Shock or Ethical Integration. Or flag that domain expertise changes the typical transition sequence.

### 6. People economically dependent on NOT adopting AI

A skilled service provider (high-end writing service, bespoke design, personalized tutoring) or worker positioned as "human-only" faces permanent economic threat from AI. Shock is not a temporary crisis to integrate — it's a permanent condition. They can't actually withdraw; livelihood is at stake. Bargaining doesn't work because the whole point is they cannot use it without destroying their business. They stay locked in Defensive Resistance permanently, not from ego defense, but because economic survival requires it.

**Fix:** Add a note that Defensive Resistance can be a permanent stable end state for users with economic incentive to stay there, distinct from identity-defense-based resistance. Or explicitly scope the model to users who can choose whether to adopt.

### 7. Pre-emptive ethical rejection (without defensive testing)

The model's path for ethical objection is: Initial Encounter → Defensive Resistance → Shock → Ethical Integration or dropout. But a user might reject AI on principle *before* ever testing it — they decide it's unethical and choose not to engage without the shock journey. Someone who says "I don't use AI because it scraped my copyrighted work without permission" hasn't gone through Defensive Resistance or Shock. Ethical Integration is described as a reflective stage *after* engagement, not before.

**Fix:** Add an explicit path from Initial Encounter directly to Ethical Integration for users with principled objections prior to tool use.

### 8. The Adoption Ceiling gap

V4 included an "Adoption Ceiling" construct — a limit on how far a user could go in the model based on values, cognitive capacity, or emotional tolerance. V5 removes it. This creates a blind spot for users stable at one intensity level but destabilized when forced higher. A manager at Maturity using AI for email summaries (low stakes) is comfortable. The company mandates AI for hiring and performance reviews (high stakes). The manager crashes back into Shock because the emotional weight of higher delegation exceeds what they've integrated. They understand AI completely. They chose not to delegate those decisions. Stability at Understanding doesn't account for choice or values.

**Fix:** Restore the Adoption Ceiling construct as a delegation ceiling tied to the person, not the tool. A user might have a delegation ceiling of 2 (can't absorb higher-stakes decisions without destabilization) even at Maturity. When forced above it, they regress predictably.

## Recommendations

1. **Scope declaration.** State explicitly in the opening section of V5 which populations the model is and is not designed to describe. Proposed text:
> This model describes adaptation of knowledge workers and creatives who encounter AI after building identity in their domain. It applies to users whose first encounter with AI happens in adulthood and whose stakes are identity-based. It does not describe AI-native users, non-knowledge-work domains, emotional-relational use, users with cognitive disabilities requiring AI as assistive technology, users already expert in the technology, or users facing permanent economic threat from adoption.

2. **Adoption Ceiling restoration.** Restore Adoption Ceiling as a delegation ceiling (level 1, 2, 3, or unbounded) tied to each person, independent of state:
> A user in Understanding or Maturity is stable only within their delegation ceiling. Forcing delegation beyond their ceiling triggers regression to Shock, even with full understanding. The ceiling is independent of state and may be modified only through sustained skill-building or readiness work.

3. **AI-native pathway.** Add an alternate entry path for users whose first meaningful exposure to AI predates or coincides with formation of domain identity:
> Users whose AI exposure comes before they form domain identity may bypass Initial Encounter and enter Understanding or Identity Expansion directly. No identity threat is present because identity is still forming. This pathway is increasingly common.

4. **Non-knowledge-work subsection.** Add a subsection describing how states and delegation experience differ in non-knowledge domains:
> In non-knowledge domains, Task Delegation maps differently: Information-level (diagnostic tools), Task Execution (automation of routine decisions). Creative and Partnership often don't apply. Shock manifests as economic threat (replacement risk) rather than identity threat (skills being replicated), making Bargaining less common and Defensive Resistance more durable.

5. **Accessibility and assistive-technology use case.** Add a note to Understanding stating capability-enabling use is out of scope:
> This model assumes the user's baseline capability exists and AI is capability-matching (replicating existing skill). Users relying on AI as assistive technology bypass Shock because the tool provides access, not competition. This use case requires separate analysis.

6. **Economic dependence (out-of-scope declaration).** Add a bullet to the Scope Declaration:
> Users whose livelihood depends on human-only service provision face permanent Shock and cannot exit the model through integration. Economic dropout is the only option. This population is out of scope.

7. **Pre-emptive ethical rejection pathway.** Add Initial Encounter → Ethical Integration as an explicit path:
> Some users perform rapid ethical appraisal at Initial Encounter and move directly to Ethical Integration or non-adoption without testing. This is distinct from post-engagement Ethical Integration and represents ethical commitment prior to capability assessment.
